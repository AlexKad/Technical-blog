Implement google cloud functions with Firebase

Recently I was setting up Firebase for one application that works with gcloud functions. It was pretty fun experience because though it was quite simple, I still faced up with some not obvious issues and had to google solutions. The main goal of this article is to keep possible issues and working solutions for this issues in one place.

  1. Create new project in google cloud.

    Go to google cloud platform console and create new project.

  2. Add bucket.

    Go to cloud storage browser and create bucket. Find more information here.

  3. Setup firebase

    Login into firebase. Go to firebase console, import your google project.

  4. Install firebase locally

    If you use node install firebase package in the root folder of your application. Run in console $ npm install firebase.

    Note if you need Firebase Storage you have to install gcloud Node.js client $ npm install --save gcloud.

  5. Init gcloud functions on the local machine.

    Run firebse init functions in console from root project directory to init Cloud Functions. Firebase will provide you with a list of your google projects and will ask you to select the default project for the current directory. Select one of the existing project or create one On the next step allo to set up all dependencies. after this instalation on your project directory you will see new folder 'functions', firebase.json file and .firebaserc file with your default project settings.

  6. Edit firebase.json

    Add "hosting: { "public": "name of your folder with index.html" }"

  7. Run firebase deploy from local console

    This will deploy your application on Firebase.

  8. Create functions

    Write functions inside newely created functions/index.js file.

  9. Solve 'cors' problems

    Install 'cors' lib.

  10. Deploy gcloud functions

    Run firebase deploy -only functions in console to deploy gcloud functions.
