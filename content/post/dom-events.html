When user somehow interacts with DOM element javascript can execute an event. Below is the list of most useful events:

Mouse events: click
contextmenu (right click)
mouseover/mouseout
mousedown/mouseup (when the mouse button pressed/released)
mousemove
Keyboard events: keydown/keyup
Form element events: submit
focus
Document events: DOMContentLoaded (when the HTML is loaded and processed, DOM is fully built)
CSS events: transitionend (when a CSS-animation finishes)

There are 3 possible ways of how assign a function (handler) for a specific event:

HTML attribute
<input value="Click me" onclick="alert('Clicked!')" type="button">
<input value="Click me" onclick="countCicks()" type="button">
DOM property
elem.onclick = function (no parentesis needed, we assign a function, not a function call)
addEventListener

onclick does not allow to assign multiple handlers to one element. It will rewrite the existing every time you assign the new handler function. addEventListener allows to add as many handlers as needed (and remove them by using removeEventListener)

elem.addEventListener( "click" , () => alert('Cool!'));

function handler2(){ alert ('this is cool too!'); }
elem.addEventListener( "click" , handler2);
